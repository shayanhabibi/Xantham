# XML Documentation Skill

You are writing F# source code xml documentation/comments.

## F# Documentation

Generate F# documentation using dotnet XML comments.

## Extensions

The extension tags described in [this RFC](https://github.com/fsharp/fslang-design/blob/main/tooling/FST-1031-xmldoc-extensions.md)
are supported.

When documenting types, it is recommended to use the `category` tag to group related types.

## Tone

The tone of the generated documentation should be clear, concise, and informative, following standard XML comment conventions.
The tone should be appropriate for the language and the audience.
